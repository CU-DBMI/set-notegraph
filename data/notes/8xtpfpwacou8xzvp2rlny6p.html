Intro


Dagger is a portable development tool implemented through Cuelang.

What's Cuelang?

CUE is an open-source data validation language and inference engine with ... many applications, such as data validation, data templating, configuration, querying, code generation and even scripting."

https://cuelang.org/docs/about/

What's Dagger?


  1. Build a .cue file
  2. Use dagger client to implement (wherever)
  3. Dagger uses buildkit to perform action
graph LR cue[ *.cue File ] --> dagger[ cmd: dagger do 'stuff' ] dagger --> buildkit buildkit --> do[perform 'stuff']

Example Usecase

  1. Build a repo .cue file for testing
  2. Test locally with dagger client
  3. Test remotely with Github Action (running the same file)
  4. Celebrate ðŸ¥³
    (because there weren't surprises with local vs remote testing!)

Installation: Dagger Cli

  • macOS: brew or curl + shell
  • Windows: psl, choco, or scoop
  • Linux: curl + shell

https://docs.dagger.io/install

Installation: Buildkit

Dagger needs a place to invoke buildkit.

  • Docker Desktop (built-in compatibility)
  • Podman (some pre-config)
  • Kubernetes (some pre-config)
  • Remotes (BYO remote environment)

https://docs.dagger.io/1223/custom-buildkit/
